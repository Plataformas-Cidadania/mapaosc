Este portal segue as diretrizes do eMAG (Modelo de Acessibilidade em Governo Eletrônico), conforme as normas do Governo Federal, em obediência ao Decreto nº 5.296, de 2.12.2004.

O termo acessibilidade significa incluir a pessoa com deficiência na participação de atividades como o uso de produtos, serviços e informações. Alguns exemplos são os prédios com rampas de acesso para cadeira de rodas e banheiros adaptados para deficientes.

Na internet, acessibilidade refere-se principalmente às recomendações do WCAG (World Content Accessibility Guide) do W3C e, no caso do Governo Brasileiro, ao eMAG (Modelo de Acessibilidade em Governo Eletrônico). O eMAG está alinhado as recomendações internacionais, mas estabelece padrões de comportamento acessível para sítios governamentais.

Na parte superior do portal existe uma barra de acessibilidade onde se encontram atalhos de navegação padronizados e a opção para alterar o contraste. Essas ferramentas estão disponíveis em todas as páginas do portal.

Atalhos de Acessibilidade

  • Alt + 1: Acessar Conteúdo
  • Alt + 2: Acessar Menu Principal
  • Alt + 3: Acessar Busca
  • Alt + 4: Acessar Rodapé

Utilização dos atalhos

Esses atalhos valem para o navegador Chrome, mas existem algumas variações para outros navegadores:

  • Quem prefere utilizar o Internet Explorer é preciso apertar o botão Enter do seu teclado após uma das combinações acima. Portanto, para chegar ao campo de busca interna é preciso pressionar Alt+3 e depois Enter.
  • No caso do Firefox, em vez de Alt + número, tecle simultaneamente Alt + Shift + número.
  • Sendo Firefox no Mac OS, em vez de Alt + Shift + número, tecle simultaneamente Ctrl + Alt + número.
  • No Opera, as teclas são Shift + Escape + número. Ao teclar apenas Shift + Escape, o usuário encontrará uma janela com todas as alternativas de ACCESSKEY da página.

Testes de Acessibilidade

Teste ASES

No teste ASES foram encontrados erros referentes a elementos do tipo link próximos entre si, elementos cuja funcionalidade se dá exclusivamente por mouse e objetos gerados por javascript que não estarão disponíveis para quem não tem javascript habilitado. Alguns avisos também foram dados em relação ao conteúdo das páginas e os mesmos foram avaliados em um teste de especialista descrito abaixo.

Teste WCAG

No teste de WCAG os seguintes erros foram encontrados para alcançar a categoria A:

  • Falta de elementos nav e/ou a utilização dos Aria Roles apropriados
  • Elementos cuja única identificação de seu estado ou função são imagens ou cores
  • Aumentar a descritibilidade das ações realizadas pelos links em seu texto, alt, title
  • Elementos que não funcionam com teclado

Um outro problema apresentado pelo teste foi referente a utilização dos Headings (Cabeçalhos), entretanto outros testes como o teste sintático do html e a verificação manual das regras do WCAG não corroboraram para existência desse problema, assim a existência do mesmo terá de ser avaliada em testes de usabilidade com usuários com deficiência.

Teste com Leitor de Tela NVDA

O teste com leitor de tela NVDA demonstrou a necessidade de alterações em diversas páginas para facilitar a navegação de quem utiliza teclado e da criação de funcionalidades extras para facilitar a navegação. É importante ressaltar que o leitor de tela DOSVOX não funciona no Mapa das OSCs devido ao fato do mesmo não processar javascript.

É importante ressaltar que os testes com leitores de tela como o DOSVOX não são possíveis posto que o funcionamento do Mapa pressupõe a execução de javascript e portanto não há de funcionar na navegação limitada do DOSVOX. Talvez haja alguma extensão ou mecanismo para o pleno funcionamento do Mapa no DOSVOX, entretanto como o mesmo não é parece existir o uso do DOSVOX é desaconselhado, sendo recomendado o uso do JAWS ou NVDA como substitutos.
